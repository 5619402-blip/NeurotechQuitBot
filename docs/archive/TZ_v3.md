# NeuroTech Quit — Финальное техническое задание
## Версия v3. Исправленная после технического аудита

| Параметр | Значение |
|---|---|
| Проект | Telegram-бот NeuroTech Quit для помощи в избавлении от никотиновой зависимости |
| Формат MVP | Telegram-бот + внешний защищённый web-плеер. Telegram Mini App не входит в MVP и остаётся на второй этап. |
| Язык интерфейса | Русский. Технические названия допускаются в скобках. |
| Главный исполнитель | Claude Code |

## 0. Инструкция для Claude Code перед началом работы

Этот документ является главным источником правды для разработки MVP. Перед написанием кода Claude Code должен прочитать документ полностью и выполнить техническую проверку.

- Работать только на русском языке.
- Не менять логику продукта без явного подтверждения пользователя.
- Сначала строить MVP, не добавлять Telegram Mini App и AI-агента поддержки.
- Ручную оплату через чек в бот не использовать как основной сценарий.
- Все экраны, кнопки, статусы и таблицы реализовывать строго по этому ТЗ.
- Если обнаружен конфликт — остановиться и задать вопрос пользователю.

## 2. Формат MVP и архитектура

MVP реализуется как связка Telegram-бота и внешнего защищённого web-плеера. Telegram Mini App не входит в первую версию.

| Компонент | Ответственность |
|---|---|
| Telegram-бот | Welcome, ролики, диагностика, согласие, выбор тарифа, статусы, follow-up, напоминания, «Мой доступ», поддержка-заглушка, отзывы и переходы. |
| Backend | Бизнес-логика, база данных, планировщик, выдача временных ссылок, обработка callback от плеера, оплата-заглушка и будущая интеграция платёжного провайдера. |
| Внешний защищённый web-плеер | Воспроизведение аудиопроцедур без скачивания, перемотки, паузы и таймлайна. Отправка server callback после полного завершения. |
| Приватное хранилище | Хранение аудиопроцедур и медиаотзывов. Должно быть доступно из РФ и через VPN. |
| Telegram Mini App | Не входит в MVP. Возможен на втором этапе для расширенного личного кабинета и красивого прогресса. |

## 3. Полный пользовательский flow

1. `/start` → Проверка статуса пользователя.
2. Welcome.
3. Смотреть объясняющий ролик или смотреть отзывы.
4. Пройти диагностику.
5. Проверка мотивации и источника решения.
6. Если мотивация низкая или человека привели — режим созревания / end_session.
7. Если мотивация 4+ и пользователь не уверен — уточняющий экран.
8. Пользовательское согласие.
9. Выбор тарифа.
10. Оплата: модуль-заглушка до подключения провайдера.
11. Экран успешной оплаты.
12. Ролик с правилами.
13. Подготовка к процедуре.
14. Экран предупреждения в плеере.
15. Процедура в защищённом плеере.
16. Server callback от плеера на backend после полного завершения.
17. Экран после процедуры.
18. Обязательные вопросы после процедуры.
19. Дальше по статусу: следующая процедура, follow-up, напоминание, 7-дневный контроль, отзыв или завершение цикла.

## 4. Возврат через /start и умная маршрутизация

При любом новом /start бот не должен вести всех на Welcome. Он обязан проверить актуальные данные пользователя и вернуть в правильную точку.

| Состояние | Действие бота |
|---|---|
| Новый пользователь | Показать Welcome. |
| Диагностика начата, но не завершена | Предложить продолжить диагностику. |
| Диагностика завершена, согласие не принято | Показать пользовательское согласие. |
| Согласие принято, оплаты нет | Показать выбор тарифа / оплату. |
| Доступ есть | Показать «Мой доступ». |
| Процедура прервана | Показать экран: «Процедура была прервана. Для корректного прохождения начните её заново». |
| Ждёт следующую процедуру | Показать «Мой доступ» с рекомендованной следующей процедурой. |
| Пользователь уже не курит | Показать экран результата / рекомендации / 7-дневный follow-up, если запланирован. |
| Протокол приостановлен | Предложить продолжить с сохранённого места. |

## 5. Экраны, кнопки и переходы

### 5.1 Welcome

Цель: познакомить пользователя с NeuroTech Quit и дать два пути: объяснение или отзывы.

- **«Смотреть объяснение»** → экран «Объясняющий ролик».
- **«Смотреть отзывы»** → раздел отзывов.

### 5.2 Объясняющий ролик

Текст: «Посмотрите короткое объяснение, как формируется никотиновая зависимость и как работает NeuroTech Quit».

- **«Смотреть»** → показать видео/медиа в Telegram.
- **«Смотреть отзывы»** → отзывы.
- **«Назад»** → Welcome.
- После просмотра/продолжения: **«Пройти диагностику»**.

### 5.3 Отзывы до диагностики

Показываются витринные отзывы: видео, аудио, текст, 2GIS. Не хранить медиа в Telegram. См. раздел об отзывах.

- **«Пройти диагностику»** → диагностика.
- **«Назад»** → Welcome.

### 5.4 Диагностика

Диагностика проводится короткими вопросами и кнопками. Шкала мотивации — кнопки 1–10, без слайдера.

**Экран 1:** имя, возраст, тип никотина, стаж.

**Экран 2:** попытки бросить, сколько раз, максимум без никотина, мотивация 1–10, источник решения.

### 5.5 Уточнение «Пока не уверен(а)»

Если motivation_level >= 4, но motivation_source = not_sure, бот не блокирует пользователя автоматически, а показывает мягкое уточнение.

- **«Я хочу продолжить»** → пользовательское согласие.
- **«Посмотреть отзывы»** → отзывы.
- **«Вернуться позже»** → экран «Сессия приостановлена».

### 5.6 Пользовательское согласие

Перед оплатой пользователь должен принять согласие. Текст согласия приведён в Приложении А. Без принятого согласия оплату и процедуру не открывать.

- **«Согласен, продолжить»** → выбор тарифа.
- **«Назад»** → предыдущий экран.

### 5.7 Выбор тарифа

**Новый пользователь** видит:
- «Одна процедура — 990 ₽»
- «Полный доступ — 4 900 ₽»

**Пользователь с access_type = single_procedure и успешной оплатой 990 ₽** видит:
- «Оплатить следующую процедуру — 990 ₽»
- «Доплатить до полного доступа — 3 910 ₽»

**Пользователь с full_access** — оплату не показывать.

Кнопку «Доплатить до полного доступа — 3 910 ₽» не показывать новым пользователям и пользователям без оплаты.

### 5.8 Оплата

Ручная оплата через чек в бот не является основным сценарием. До подключения платёжного провайдера используется тестовая заглушка оплаты.

После выбора тарифа показывать экран оплаты.

**До подключения провайдера текст:**
«Оплата временно находится в тестовом режиме. Сейчас можно проверить весь путь прохождения без реального списания средств».

**Кнопки:** «Тестово открыть доступ», «Назад к выбору тарифа».

При нажатии «Тестово открыть доступ»: создать payment со status = paid, открыть выбранный доступ и показать экран успешной оплаты.

После подключения провайдера заглушка заменяется на реальную оплату. Логика открытия доступа остаётся той же.

### 5.9 Успешная / неуспешная оплата

| Экран | Текст | Кнопки / переходы |
|---|---|---|
| Оплата прошла успешно | Оплата прошла успешно. Доступ открыт. Перед запуском процедуры посмотрите короткие правила прохождения. Это поможет пройти её правильно и без лишних ошибок. | «Перейти к правилам» → ролик с правилами. |
| Оплата не прошла | Оплата не прошла. Возможно, платёж был отменён или возникла ошибка банка. Попробуйте ещё раз или выберите другой способ оплаты. | «Повторить оплату», «Выбрать другой тариф», «Назад». |

### 5.10 Ролик с правилами

Видео показывается как медиа в Telegram. Не требовать обязательного отслеживания досмотра до конца. Сохранять события: «Смотреть» и «Продолжить».

Текст: «Перед процедурой посмотрите короткие правила. Это важно для корректного прохождения».

**Кнопки:** «Смотреть», «Перейти к подготовке», «Назад».

### 5.11 Сессия приостановлена (end_session)

Экран показывается, когда пользователь выбирает вернуться позже или не готов продолжать.

Текст: «Хорошо. Вы можете вернуться к NeuroTech Quit позже. Когда будете готовы, просто откройте бот снова — мы продолжим с того места, где вы остановились».

**Кнопки:** «Вернуться в главное меню», «Пройти диагностику заново».

В базе обязательно сохранить: user_status = protocol_paused, текущий этап, причину выхода (если есть), дату выхода.

## 6. Подготовка к процедуре

Общий экран называется «Подготовка к процедуре». Для первой процедуры используется расширенная версия. Для следующих процедур используется сокращённая версия.

### 6.1 Расширенная подготовка перед первой процедурой

Порядок пунктов фиксированный. Не менять местами.

1. Последний контакт с никотином, предусмотренный протоколом, завершён непосредственно перед процедурой. Не заранее за час или два, а именно перед началом сессии, как прощальный ритуал.
2. Пользователь собрал и выбросил все курительные принадлежности в доме: сигареты, стики, вейпы, нагреватели, зажигалки, спички и всё, что может вернуть старый сценарий.
3. В доме не осталось курительных триггеров.
4. Пользователь в трезвом и спокойном состоянии.
5. Пользователь выбрал спокойное место, желательно тёмную комнату, где его никто не будет отвлекать.
6. Телефон выключен или убран. Уведомления и внешние раздражители отключены.
7. Пользователь сидит удобно. Не лежит. Желательно в кресле или на стуле, в устойчивом спокойном положении.
8. Наушники надеты.
9. Пользователь готов пройти процедуру полностью и без остановок.

**Кнопки:** «Да, я готов», «Мне нужно подготовиться», «Назад».

### 6.2 Сокращённая подготовка перед следующими процедурами

1. Пользователь в спокойном месте.
2. Телефон убран, уведомления отключены.
3. Пользователь сидит удобно, не лежит.
4. Наушники надеты.
5. Пользователь готов пройти процедуру полностью и без остановок.

**Кнопки:** «Да, я готов», «Мне нужно подготовиться», «Назад».

## 7. Внешний защищённый web-плеер

- Плеер не покупается как отдельный сервис. Он разрабатывается как часть проекта.
- Аудиофайлы не хранятся в Telegram и не отправляются пользователю.
- Аудио лежит во внешнем приватном хранилище, доступном из РФ и через VPN: Яндекс Облако / Selectel / VK Cloud / Timeweb Cloud или аналог.
- Размер аудиофайлов разный, ориентировочно 100–200 МБ каждый.
- Обязательно сохранять stereo L/R. Нельзя переводить в mono или применять обработку, которая ломает бинауральные ритмы.
- Доступ к плееру только по временной защищённой ссылке / токену.
- Файл нельзя скачивать напрямую. Не давать прямую ссылку на аудио.

### 7.1 Экран перед запуском аудио

Перед запуском пользователь видит предупреждение:

«Процедуру необходимо проходить полностью и без остановок. Не ставьте аудио на паузу, не выходите из процедуры и не отвлекайтесь. Последовательность аудио и инструкций имеет значение для корректной работы метода. Если процедура была прервана, остановлена или пользователь отвлекался, NeuroTech не может гарантировать корректность результата этой сессии. В таком случае процедуру рекомендуется пройти заново с самого начала».

**Кнопки:** «Начать процедуру», «Вернуться назад».

### 7.2 Во время процедуры

- Нет паузы.
- Нет перемотки.
- Нет таймлайна.
- Нет скачивания.
- Нет перехода дальше до завершения.
- Единственная кнопка: **«Экстренно выйти»**.

### 7.3 Экстренный выход

Текст: «Процедура была прервана. Для корректного результата её нужно будет пройти заново с самого начала».

**Кнопка:** «Начать заново позже».

Процедура не засчитывается как завершённая: session_status = interrupted, is_counted_as_completed = false.

### 7.4 Механизм «плеер → backend → бот»

1. После полного завершения аудио внешний плеер отправляет **server callback** на backend.
2. Callback содержит procedure_session_id и событие completed.
3. Backend меняет session_status на completed, сохраняет completed_at.
4. Бот видит обновлённый статус и показывает экран «После процедуры / фиксация состояния».

Механизм не должен зависеть от Telegram WebApp API и должен работать через обычный браузер / Telegram WebView.

## 8. Порядок процедур и доступ

- **Базовый минимум:** Анти-табак → Быстрый рычаг → Анти-табак.
- Если пользователю нужно продолжать, дальше чередование: Быстрый рычаг → Анти-табак → Быстрый рычаг → Анти-табак и так далее.
- **Альфа-процедура** является дополнительной поддерживающей процедурой. Она не меняет current_step_number и не ломает основное чередование.
- До завершения базового протокола пользователь не выбирает порядок сам. Бот ведёт его по очереди.

### 8.1 Тариф 990 ₽: single_procedure

После успешной оплаты single_procedure открыть одну основную процедуру anti_tobacco и одну поддерживающую процедуру alpha. Это не full_access.

Поля в access_rights:
- access_type = single_procedure
- paid_main_procedures_count = 1
- available_alpha_sessions_count = 1
- used_main_procedures_count = 0
- used_alpha_sessions_count = 0
- upgrade_available = true
- upgrade_amount = 3910

После прохождения первой основной процедуры следующая основная процедура не открывается бесплатно.

Показывать: «Оплатить следующую процедуру — 990 ₽» или «Доплатить до полного доступа — 3 910 ₽».

Альфа в тарифе 990 ₽ доступна один раз и не меняет current_step_number.

## 9. «Мой доступ»

«Мой доступ» — динамический экран. Он строится на основе данных пользователя, а не является статичным меню.

| Условие | Что показывать |
|---|---|
| access_type = full_access | Полный доступ, «Продолжить протокол», «Мой прогресс», «Поддержка». |
| access_type = single_procedure | Оплаченная процедура, «Оплатить следующую процедуру — 990 ₽», «Доплатить до полного доступа — 3 910 ₽», «Мой прогресс». |
| procedure interrupted | Показать, что процедура была прервана, и предложить начать заново. Продолжать с места остановки нельзя. |
| следующая процедура доступна | Показать следующую рекомендованную процедуру и кнопку «Продолжить протокол». |
| доступа нет | «Мой доступ» не показывать, вести на Welcome / диагностику / оплату. |

### 9.1 Кнопка «Продолжить протокол»

Кнопка не ведёт на один статичный экран. Она запускает умную проверку статуса пользователя.

- Если есть незавершённая/прерванная процедура → экран «Процедура была прервана / начать заново».
- Если следующая процедура доступна → экран подготовки к процедуре.
- Если нужна оплата → выбор тарифа / оплата.
- Если пользователь прошёл 3+ процедур → вопросы результата / follow-up.
- Если цикл завершён → экран поздравления / рекомендаций.

## 10. После процедуры / фиксация состояния

### 10.1 Экран сразу после процедуры

После server callback и статуса completed бот показывает экран инструкции. Вопросы идут следующим экраном.

Текстовый смысл экрана:
- процедура завершена; поздравление;
- в ближайшее время наблюдать состояние;
- воздерживаться от никотина;
- не приближаться к триггерам;
- не проверять себя;
- если тяга появится, это нормально;
- при тяге вспомнить заключительную эмоциональную часть процедуры / нейродинамической медитации и воздержаться от курения.

**Кнопка:** «Далее» → обязательные вопросы после процедуры.

### 10.2 Обязательные вопросы после процедуры

- Вопросы после процедуры обязательны.
- Кнопку «Зафиксировать результат» убрать.
- Ответы сохраняются автоматически после каждого ответа.
- Нельзя пропустить вопросы.
- Нельзя перейти к следующей процедуре, оплате, follow-up или другому этапу без завершения обязательных вопросов.
- После ответа на все вопросы показывать кнопку «Продолжить».

**Минимальный набор вопросов:**
1. Как вы себя чувствуете?
2. Есть ли тяга?
3. Насколько сильная тяга?
4. Курили ли вы после процедуры?
5. Нужна ли ещё процедура?
6. Нужна ли помощь?

## 11. Follow-up и напоминания

### 11.1 Follow-up на следующий день

Отправляется автоматически через backend-планировщик. Состоит из 2 экранов.

| Экран | Вопросы / кнопки |
|---|---|
| Экран 1: Состояние и курение | 1) Курили ли вы после процедуры? — Нет, не курил(а) / Один раз / Несколько раз / Да, вернулся(ась) к курению.<br>2) Есть ли сейчас желание курить? — Нет / Слабое / Среднее / Сильное.<br>3) Стало ли вам легче воздерживаться? — Да, заметно легче / Немного легче / Пока не понимаю / Нет, пока так же. |
| Экран 2: Триггеры и продолжение | 4) В каких ситуациях тяга появлялась сильнее всего? — Утром / После еды / С кофе / В стрессе / За рулём / Когда рядом курили / Тяги почти не было.<br>5) Что заметили? — Больше спокойствия / Меньше тяги / Меньше автоматизма / Появилось сопротивление курению / Пока ничего не заметил(а) / Стало тревожнее.<br>6) Хотите продолжить протокол? — Да, продолжить / Я уже не курю / Не хочу продолжать. |

Кнопка «Пока хочу понаблюдать» исключена.

В next_day_followups обязательно добавить scheduled_at и sent_at.

### 11.2 Напоминания о следующей процедуре

- После завершения каждой основной процедуры бот создаёт напоминание через **48 часов** о следующей положенной процедуре.
- Перед отправкой напоминания backend обязан проверить актуальный статус пользователя.
- Если следующая процедура уже пройдена — напоминание не отправлять / отменить.
- После прохождения новой процедуры создать новое напоминание от даты последней завершённой процедуры.
- Максимум 3 напоминания по одной следующей процедуре.

Текст: «По протоколу NeuroTech вам рекомендуется пройти следующую процедуру: [название процедуры]. Это следующий шаг для закрепления результата и продолжения работы с зависимыми реакциями».

**Кнопки:** «Пройти следующую процедуру», «Я уже бросил курить», «Не хочу продолжать».

### 11.3 Follow-up через 7 дней

Создаётся, если пользователь ответил «Я уже не курю» / «Да, не курю».

Через 7 дней бот задаёт короткий опрос: курил ли за последние 7 дней, есть ли желание курить, что изменилось, нужна ли поддержка дальше.

Если пользователь не курит и тяги нет / она слабая, бот отправляет красивое поздравление и предлагает оставить отзыв.

**Кнопки:** «Оставить отзыв», «Отправить другу», «Пройти Альфа-процедуру», «Завершить цикл».

trigger_source значения: after_procedure_not_smoking, next_day_not_smoking, final_check_not_smoking, manual_admin.

### 11.4 Планировщик

- Для MVP использовать **node-cron** + таблица reminders.
- Backend регулярно проверяет, какие сообщения пора отправить.
- Перед отправкой проверять актуальный статус.
- Если пользователь заблокировал бота или сообщение не доставлено — фиксировать ошибку.
- При росте нагрузки можно заменить на BullMQ / Redis.

## 14. Отзывы: хранение, загрузка и модерация

- Отзывы не хранить в Telegram. Медиаотзывы и превью хранить во внешнем хранилище. Текстовые отзывы хранить в базе.
- Витринные отзывы показываются новым пользователям до диагностики.
- Типы витринных отзывов: video, audio, text, 2gis.
- Видео/аудио/превью хранить во внешнем хранилище. Для MVP можно Supabase Storage или то же приватное хранилище, выбранное для медиа.
- Отзывы пользователей после результата хранить отдельно от витринных отзывов.
- Новые пользовательские отзывы сначала имеют moderation_status = pending, затем approved или rejected.
- Администратор должен иметь возможность добавить, отключить, отсортировать витринные отзывы и модерировать пользовательские отзывы.
