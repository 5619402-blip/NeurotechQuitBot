## 0. Краткое резюме проекта

NeuroTech Quit — Telegram-бот для помощи в избавлении от никотиновой зависимости через технологию NeuroTech, диагностику готовности, оплату доступа, аудио-процедуры, поддержку, напоминания и контроль результата.

**Ключевой принцип MVP:** не усложнять. Сначала собрать рабочий бот с понятным flow, оплатой, защищённым аудиоплеером, сохранением прогресса, базовой поддержкой и напоминаниями.

---

## 1. Концепция продукта

NeuroTech Quit — Telegram-бот для помощи в избавлении от никотиновой зависимости.

**Главная идея продукта:**
Не просто "терпеть и держаться", а мягко ослаблять автоматизм, который запускает тягу к никотину, и в итоге убрать эту зависимость из своей жизни.

---

## 2. Целевая аудитория

Взрослые пользователи с никотиновой зависимостью:
- Сигареты
- Айкос
- Вейпы
- Смешанное употребление

Основное ядро аудитории: 25–45 лет.

---

## 3. Позиционирование

NeuroTech Quit позиционируется **не** как сила воли, мотивация или страшилки про курение.

Позиционируется как:
- технология помощи в избавлении от никотиновой зависимости
- система работы с автоматическими реакциями мозга и зависимыми паттернами

**Основной акцент:**
- работа с двумя корнями зависимости: физиологическим и психологическим
- автоматическими ритуалами, внутренними триггерами, подсознательными реакциями

**Главная идея:** зависимость — это закреплённая нейронная сеть автоматических реакций. Технология NeuroTech помогает ослаблять старые зависимые связи и формировать новые, более здоровые реакции.

## 4. Актуальная архитектура MVP

> Важно: объясняющий ролик 2:20 находится **ДО** диагностики, на первом экране Welcome. После ролика появляется основной CTA «Пройти диагностику».

### Flow экранов:
1. Welcome / первый экран
2. Смотреть объяснение / ролик 2:20
3. CTA «Пройти диагностику»
4. Смотреть отзывы (опциональный путь)
5. Диагностика, экран 1
6. Диагностика, экран 2
7. Проверка мотивации и источника решения
   - Низкая мотивация → режим созревания
   - Допуск → пользовательское согласие
8. Экран оплаты
9. Ролик с правилами
10. Подготовка к первой процедуре
11. Процедура 1: Анти-табак
12. Экран после процедуры
13. Follow-up и напоминание о следующей процедуре
14. Процедура 2: Быстрый рычаг
15. Процедура 3: Анти-табак
16. Итоговая проверка результата
17. Поздравление / рекомендации / отзыв / отправить другу
18. Поддержка / AI-агент во втором этапе

### Правильный flow (для кода):
```
welcome -> intro_video -> questionnaire_step_1 -> questionnaire_step_2
-> low_motivation OR user_consent -> payment -> rules_video
-> preparation_screen -> procedure_1 -> after_procedure
-> reminders -> next_procedures -> final_check -> final_success
```

## 5. Общие правила разработки

- Интерфейсные названия, кнопки и сообщения пользователю писать на **русском языке**.
- Не добавлять лишние экраны без необходимости.
- **Не отправлять аудиофайлы пользователю в Telegram.**
- Все аудио-процедуры открываются только через защищённый плеер.
- Пользователь может выйти на любом этапе и вернуться к месту остановки.
- Все важные действия пользователя логируются в таблицу событий.
- AI-агент поддержки — не обязательный для первой сборки MVP, можно заменить заранее прописанными ответами.

## 19. Аудио-процедуры и медиа-логика

Процедуры используются как аудио, без видео-дорожки.

**Технические требования к файлам:**
- Формат: M4A
- Стерео L/R, 48 000 Гц, битрейт ~250 кбит/с
- Размер первой процедуры ~116 МБ

**Правила:**
- Аудио должно оставаться стерео, левый и правый каналы **не смешивать**.
- Не включать шумоподавление, автоулучшение голоса, нормализацию, перевод в моно.
- Бинауральные ритмы требуют сохранения Stereo L/R.
- **Файлы не отправляются пользователю в Telegram.**
- Воспроизведение только через защищённый плеер.

| Поле | Значение / требование |
|---|---|
| screen_id | procedure_1_player |
| Назначение | Запуск первой процедуры «Анти-табак». |
| Проверка доступа | Проверить payment_status = paid и access_type = single_procedure или full_access. |

**Функции плеера:** Play, pause, продолжить с места остановки, начать заново, мобильная поддержка, стерео-звук.

**Сохранять:** `procedure_session_id`, `procedure_id`, `started_at`, `audio_position`, `completed_at`, `status`

После завершения → `after_procedure_screen`

| Повторный доступ | Если пользователь имеет доступ, при каждом запуске генерировать новую временную ссылку. |
| Возврат | Если пользователь прервал аудио, предложить «Продолжить с места остановки» или «Начать процедуру заново». |

## 30. Сохранение прогресса и возврат пользователя

Пользователь может выйти на любом этапе. При возвращении бот показывает: «Вы остановились здесь. Продолжить?»

| Ситуация | Что делает бот |
|---|---|
| Вышел во время диагностики | Сохраняет ответы и возвращает на тот же шаг. |
| Вышел после оплаты, но до процедур | Возвращает к подготовке или запуску процедуры. |
| Прервал аудио | Предлагает «Продолжить с места остановки» или «Начать процедуру заново». |
| Не ответил на вопросы после процедуры | Возвращает к экрану фиксации состояния. |
| Между процедурами | Показывает текущий статус, следующую процедуру и рекомендованное время. |

**Кнопки при возврате:** Продолжить, Начать заново, Главное меню

**Обязательные поля состояния:** `current_screen`, `current_step`, `procedure_id`, `audio_position`, `payment_status`, `access_type`, `last_activity_at`

## 36. Рекомендации технической консультационной группы

- Не хранить процедуры в Telegram.
- Использовать приватное хранилище и защищённый плеер.
- Для РФ выбирать хранилище/CDN, стабильно доступное из России.
- Везде сохранять `current_screen` и статус пользователя.
- Разделить оплату 990 ₽ и 4 900 ₽ через `access_type`.
- AI-агента не делать в MVP как обязательный функционал.
- Сначала собрать стабильный минимальный путь: ролик → диагностика → согласие → оплата → правила → процедура → follow-up.
