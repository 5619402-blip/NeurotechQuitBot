# NeuroTech Quit — память проекта

NeuroTech Quit — Telegram-бот + внешний защищённый плеер для прохождения протокола избавления от никотиновой зависимости.

## Роли инструментов

Codex:
- технический архитектор;
- ревьюер;
- постановщик задач для Claude Code;
- проверяет риски;
- пишет точные промты для Claude Code;
- не должен менять код без прямой команды.

Claude Code:
- исполнитель в коде;
- читает ТЗ;
- вносит изменения;
- запускает проверки;
- исправляет ошибки.

## MVP

В MVP входит:
- Telegram-бот;
- диагностика;
- оплата в тестовом режиме;
- пользовательское согласие;
- порядок процедур;
- защищённый внешний плеер;
- фиксация состояния после процедуры;
- напоминания;
- follow-up;
- поддержка;
- отзывы.

Telegram Mini App пока не входит в MVP.

## Стартовый flow

Welcome → вводный ролик 2:20 → Пройти диагностику → questionnaire_step_1

На первом экране:
- короткое знакомство с NeuroTech Quit;
- кнопка «Смотреть объяснение»;
- кнопка «Смотреть отзывы»;
- после просмотра ролика CTA «Пройти диагностику».

## Диагностика и допуск

Оплата показывается после успешной диагностики и допуска.

Условия допуска:
- motivation_level >= 4;
- пользователь подтвердил личное решение.

Если пользователь отвечает «Пока не уверен(а)» при мотивации 4+, показывается уточняющий экран:
- «Я хочу продолжить» → пользовательское согласие;
- «Посмотреть отзывы» → отзывы;
- «Вернуться позже» → сессия приостановлена.

## Пользовательское согласие

Пользователь подтверждает:
- добровольное прохождение;
- сервис не является медицинской услугой;
- сервис не заменяет врача, психотерапевта или лечение;
- результат индивидуален и не гарантируется;
- пользователь находится в трезвом и спокойном состоянии;
- пользователь согласен с правилами прохождения;
- аудиофайлы не передаются пользователю и доступны только через защищённый плеер;
- пользователь согласен на обработку данных внутри сервиса;
- при прерывании процедуры результат сессии не гарантируется.

## Тарифы

Новый пользователь видит выбор тарифа:
- «Одна процедура — 990 ₽»
- «Полный доступ — 4 900 ₽»

Если пользователь оплатил одну процедуру за 990 ₽:
- после первой процедуры или при попытке открыть следующую процедуру показывать:
  - «Оплатить следующую процедуру — 990 ₽»
  - «Доплатить до полного доступа — 3 910 ₽»

Если пользователь оплатил полный доступ 4 900 ₽:
- оплату больше не показывать;
- все процедуры, повторные прохождения и поддержка доступны в рамках полного доступа.

## Тестовая оплата

До подключения платёжного провайдера оплата работает в тестовом режиме.

Текст:
«Оплата временно находится в тестовом режиме. Сейчас можно проверить весь путь прохождения без реального списания средств».

Кнопки:
- «Тестово открыть доступ»
- «Назад к выбору тарифа»

При нажатии «Тестово открыть доступ»:
- создать тестовую оплату со статусом paid;
- открыть выбранный доступ;
- перевести пользователя на экран «Оплата прошла успешно».

## Порядок процедур

Базовый минимум:
1. «Анти-табак»
2. «Быстрый рычаг»
3. «Анти-табак»

После базового минимума:
- «Быстрый рычаг»
- «Анти-табак»
- «Быстрый рычаг»
- «Анти-табак»
- далее по чередованию.

«Альфа-процедура» — дополнительная поддерживающая процедура.
Она может быть доступна между основными процедурами или по состоянию пользователя, но не меняет основной порядок.

До завершения базового протокола пользователь не выбирает порядок сам. Бот ведёт его по очереди.

## Защищённый плеер

Пользователь нажимает:
- «Перейти к процедуре»
- или «Продолжить с места остановки»

Бот проверяет:
- доступ;
- текущую разрешённую процедуру;
- порядок протокола;
- наличие незавершённой процедуры.

После проверки бот создаёт временную защищённую ссылку на внешний плеер.

Аудиофайл:
- не отправляется в Telegram;                 - не скачивается пользователем напрямую;
- должен сохранять stereo L/R;
- открывается только через защищённый плеер.

В плеере:
- нет паузы;
- нет перемотки;
- нет таймлайна;
- нет скачивания;
- нет прямой ссылки на файл;
- нет перехода дальше до завершения.

Единственная кнопка во время процедуры:
- «Экстренно выйти»

Если пользователь нажал «Экстренно выйти»:
- процедура не засчитывается;
- статус не становится completed;
- при возврате доступно только начать эту процедуру заново.

## После процедуры

После завершения процедуры пользователь должен перейти на экран фиксации состояния.

Шаг 11 должен:
- показать экран после процедуры;
- задать обязательные вопросы Q1–Q6;
- сохранить ответы;
- определить следующий сценарий.

Если пользователь просит поддержку:
- показать экран «Поддержка NeuroTech»;
- попросить описать вопрос одним сообщением;
- сохранить обращение;
- отправить его в админ-чат;
- показать кнопки «Мой доступ» и «Главное меню».

## Напоминания

После завершения каждой основной процедуры бот ставит напоминание через 48 часов.

Текст:
«По протоколу NeuroTech вам рекомендуется пройти следующую процедуру: [название процедуры]. Это следующий шаг для закрепления результата и продолжения работы с зависимыми реакциями».

Кнопки:
- «Пройти следующую процедуру»
- «Я уже бросил курить»
- «Не хочу продолжать»

Если пользователь ничего не нажимает:
- отправить максимум ещё 2 повторных напоминания;
- всего максимум 3 напоминания по одной следующей процедуре.

Перед отправкой бот всегда проверяет актуальный статус:
- если следующая процедура уже пройдена — напоминание не отправлять;
- если процедура ещё не пройдена и лимит не исчерпан — отправить.

## Follow-up

Если пользователь отвечает «Я уже не курю» / «Да, не курю», через 7 дней отправить короткий опрос:
- курил ли за последние 7 дней;
- есть ли желание курить;
- что изменилось;
- нужна ли поддержка дальше.

Если пользователь не курит и тяги нет или она слабая:
- отправить поздравление;
- предложить оставить отзыв.

Кнопки после поздравления:
- «Оставить отзыв»
- «Отправить другу»
- «Пройти Альфа-процедуру»
- «Завершить цикл»

## Отзывы

Отзывы не хранить в Telegram.

Витринные отзывы:
- видео/аудио/превью хранить во внешнем хранилище;
- текстовые отзывы хранить в базе.

Таблица reviews:
- тип отзыва: video / audio / text / 2gis;
- имя клиента;
- заголовок;
- описание;
- ссылка на файл;
- превью;
- источник;
- активен / не активен;
- порядок сортировки.

Пользовательские отзывы хранить отдельно в user_reviews:
- сначала статус pending;
- затем approved / rejected.

- Шаги 1–10.

- Защиту при getProcedureById === null.
- Логику доступа.
- Логику оплаты.
- protocol_progress.
- Порядок основных процедур.
- Правило: Альфа-процедура не меняет основной порядок.
- Правило: прерванная процедура не засчитывается.
