# Следующая задача — Шаг 11

Нужно реализовать экран после процедуры и обязательные вопросы Q1–Q6.

## Контекст

Шаги 1–10 уже реализованы.
Шаг 10 прошёл аудит.
Код Шага 11 ещё не реализован.

После завершения процедуры пользователь должен перейти на экран фиксации состояния после процедуры.

## Цель Шага 11

После процедуры бот должен:
1. зафиксировать факт завершения процедуры;
2. показать экран после процедуры;
3. задать обязательные вопросы Q1–Q6;
4. сохранить ответы в базе;
5. в зависимости от ответов перевести пользователя в правильный следующий сценарий:
   - следующий шаг протокола;
   - напоминание через 48 часов;
   - follow-up через 7 дней;
   - поддержка;
   - завершение / пауза.

## Обязательные требования

- Не списывать доступ повторно, если процедура невалидная.
- Не обновлять protocol_progress, если getProcedureById вернул null.
- Не засчитывать прерванную процедуру.
- Основная процедура засчитывается только после корректного завершения в плеере.
- Альфа-процедура не меняет current_step_number.
- Основные процедуры меняют current_step_number по правилам протокола.
- Вопросы Q1–Q6 должны быть обязательными.
- Ответы нужно сохранять в базе.
- После завершения основных процедур нужно создавать напоминание через 48 часов.
- Если пользователь сообщает, что уже не курит, нужно запланировать follow-up через 7 дней.
- Если пользователь просит поддержку, нужно использовать MVP-логику поддержки: сохранить обращение и отправить в админ-чат.

## Важное решение по спорному месту

Если Q5 = «Пока не знаю» и Q6 = «Да, нужна поддержка», временно использовать MVP-логику поддержки:
- показать экран «Поддержка NeuroTech»;
- попросить пользователя описать вопрос одним сообщением;
- сохранить обращение;
- отправить в админ-чат;
- после отправки показать кнопки «Мой доступ» и «Главное меню».

Не делать сложный сценарий поддержки в Шаге 11.
